# BigAmbitionsMP — Structural Audit (2026-06-24)

Read-only multi-agent audit: 48 agents, ~3.95M tokens. **37 raw findings → 30 adversarially verified → 24 distinct** after dedup. No crashes or unrecoverable corruption found. Worst real-world cases: permanent un-buyability of fresh-game buildings, silent character loss on checkpoint load, and cross-machine economy/ownership desync. The architecture's invariants are sound — the bugs are **omissions at the seams** (session teardown + inbound-authority validation applied inconsistently across modules).

Status: **findings only — nothing fixed off this list yet.** Each fix is re-confirmed against current code before applying.

## Cross-cutting root themes (fix the class, not just the instance)

- **(a) Session-boundary teardown is inconsistent.** Static/process-lifetime state not reset at `Start()`/`Stop()`/scene-reset while its siblings are: `BuildingRealEstateOwners`, all 6 `InteriorSync` fields, `MPAudit._streaks`, `_characterNamesByPlayerId`, `_clientSelfStats`, `_correctionHours`. One "every static collection vs the reset blocks" pass catches the whole class.
- **(b) Inbound-authority validation fails open.** `BusinessChange`, `RetailPrices`, `ListForSale`, `CancelSale`, `SaleCompleted`, `RegisterCashier` all short-circuit to "allowed" when the building has no `BuildingOwners` entry, and none consult `BuildingRealEstateOwners`. A shared `SenderOwns(addr, senderPid)` helper (checks both ledgers) applied uniformly is the durable fix.
- **(c) Join-replay completeness (Class 4).** `ManualPaused`, player appearances, and (transiently) `OwnerOpenState` are host-authoritative but absent from `SendJoinReplayTo`.

## HIGH — 4 (all trivial fixes, clear priority)

### H1 · `BuildingRealEstateOwners` leaks across a new hosting session → fresh-world buildings permanently un-buyable
`MPServer.cs:378`. `Start()` clears `BuildingOwners` + `CashByStableId` but never the bought-real-estate ledger (only the load path clears it). After exit → Start New Game, stale session-A owners survive, so `HandleBuyRequest` denies the buy — a brand-new-world building is permanently un-buyable, wrongly AI-shielded, and marked unavailable on every client.
**Fix:** add `BuildingRealEstateOwners.Clear();` right after line 378 (symmetric with the existing clear).

### H2 · `InteriorSync.Reset()` is dead code → prior session's interior served as `Authoritative=true` (Class-8 overwrite)
`InteriorSync.cs:48`. `Reset()` clears all 6 per-session fields but has **zero call sites** (every sibling sync module wires its Reset). Worst path: a cached `Authoritative=true` snapshot from session A survives a re-host/load and overwrites session B's real building; stale hashes also suppress the correct push.
**Fix:** wire `InteriorSync.Reset()` into the per-session reset block (alongside `PassengerSync.Reset()`) and/or `MPServer.Stop()`/`MPClient.Disconnect()`. Method already exists, idempotent.

### H3 · `CarryForwardAbsentMembers` filters on `"guid-"` only → every Steam player excluded → silent character loss
`MPSaveCoordinator.cs:803`. StableId is `steam-<id>` for Steam players (the normal case) and `guid-` only as fallback; the filter rejects every Steam folder, so the carry-forward never runs for them. On an `-auto`/`-disconnect` checkpoint with an offline Steam member, that member is fresh-started on resume — the exact loss the 2026-06-23 carry-forward was meant to prevent.
**Fix:** `if (!stable.StartsWith("guid-") && !stable.StartsWith("steam-")) continue;` (or drop the prefix test and rely on the existing `NewestHsg==null` + `connected.Contains` skips).

### H4 · `BuildingRealEstateOwners` not re-keyed to live pid on reconnect → bought property stuck under stableId
`MPServer.cs:2496`. Re-host reserves absent owners under stableId in **both** ledgers, but the reconnect-reclaim re-keys only `BuildingOwners`. A reconnecting real-estate buyer keeps the stableId entry forever → their own building repeatedly marked unavailable, and buy-arbitration no longer recognizes them as owner.
**Fix:** add the symmetric re-key loop for `BuildingRealEstateOwners` after the `BuildingOwners` loop; extend the orphan diagnostic too.

## MEDIUM — 12

- **M1 · `HandleSaleCompleted` no sender/owner check** — `MPServer.cs:2182`. Unconditional `BuildingRealEstateOwners.TryRemove` with no `SenderIs`/owner check (its 3 siblings all validate) → client-forgeable ownership wipe/steal. **Fix:** add `SenderIs` + gate removal on `cur == senderPid || cur == ""`.
- **M2 · `ListForSale`/`CancelSale` no sender/owner check** — `MPServer.cs:2162`. Mutate authoritative `gi.buildingsForSale` with only a non-empty AddressKey check → forged market entries (→ unauthorized transfer) / de-listing griefing. **Fix:** gate both on sender owning the real-estate ledger entry; `SenderIs` on CancelSale.
- **M3 · `BusinessChange` guard fails open** — `MPServer.cs:951`. Short-circuits to allowed for AI/unowned; ignores `BuildingRealEstateOwners` → client forges name/type/sign/hours onto AI buildings, broadcast + persisted. **Fix:** fail closed — require `o == senderPid` (or real-estate equivalent).
- **M4 · `RetailPrices` guard fails open** — `MPServer.cs:1049`. Same shape → forged authoritative prices cached + replayed to late joiners, skewing the AI price sim. **Fix:** require positive ownership match; consult both ledgers.
- **M5 · Manual-pause absent from join replay** — `MPServer.cs:1523`. A (re)joiner during a shared pause runs at 1x while others are frozen; a later unpause no-ops for them. **Fix:** idempotent peer-targeted `if (TimeSync.ManualPaused) Send(peer, ManualPause{true})` in `SendJoinReplayTo`.
- **M6 · `ResumeFromDisconnectPause` clears a concurrent deliberate pause** — `MPServer.cs:337`. Both pause causes share one flag; reconnect unconditionally unpauses, cancelling a deliberate pause nobody lifted. **Fix:** track the two pause causes separately.
- **M7 · Synthetic `dailyIncomes` baked into client saves** — `GameStatePatcher.cs:674`. `dailyIncomes` (a save field) is overwritten with synthetic data on clients and not stripped at the save boundary → fabricated income history persists into `.hsg`, visible on SP load. **Fix:** strip/restore at the save choke point, or drive the graph from the runtime-only `RivalState` history.
- **M8 · `RunDaily` orphan-rivalState purge unguarded** — `MPPatches.cs:2454`. `RemoveAll` predicate calls IL2CPP id-lookup with no per-entry try/catch (its sibling has one); a throw aborts the client day-roll, and a transiently-unresolved real rival's history is deleted. **Fix:** wrap the lookup in try/catch; restrict the destructive purge to the host.
- **M9 · `TimeSync` clock catch-up not gated / not cleared** — `TimeSync.cs:182`. `TickClockCorrection` runs unconditionally (its AheadFreeze counterpart is gated) and `_correctionHours` is never cleared on session-end → residual catch-up fast-forwards the single-player fork after disconnect. **Fix:** gate on `MPServer.IsRunning || MPClient.InMpGame`; clear on session-end.
- **M10 · Passenger seat not released on disconnect** — `MPServer.cs:635`. `OnPeerDisconnected` never calls `PassengerSync.ApplyExit` → phantom seat blocks boarding on an empty car and leaks into join replay. **Fix:** marshal `ApplyExit(leftPlayer)` + release riders of the departing player's vehicles + broadcast `PassengerExit`; mirror client-side.
- **M11 · Time-skip vote not removed on disconnect** — `MPRestSync.cs:514`. A dropped voter's stale vote stays in `_hostVotes` while `required` drops → consensus stuck true, remaining player can't stop the skip. **Fix:** `MPRestSync.RemovePlayer(pid)` (`_hostVotes.Remove` + rebroadcast) from `OnPeerDisconnected`.
- **M12 · Restored cash of exactly $0 / overdraft never re-applied** — `MPSaveCoordinator.cs:542`. The cash overlay gates on `money>0` (and the manifest seed on `!= 0`), so a $0/negative authoritative balance never corrects a stale loaded balance → player shown/able to spend phantom money. **Fix:** use a `float?`/`bool hasPending` sentinel instead of overloading 0.

## LOW — 9

- **L1 · `InteriorOwnerSnapshot` applies prices with no sanity gate** — `InteriorSync.cs:201`. The owner-snapshot path skips the Count>500 / `IsSaneMoney` checks the dedicated `RetailPrices` handler applies → NaN/negative prices reach the sim; unbounded `ItemInstances`. **Fix:** share the validation predicate across both channels; cap item count.
- **L2 · Host rival/self-stat caches persist across re-host** — `MPServer.cs:286`. `_characterNamesByPlayerId` + `_clientSelfStats` never cleared → stale rival-AI targeting/name on a returning player. **Fix:** `.Clear()` both in the `Start()` reset block.
- **L3 · `BuildFullSnapshot` re-seeds shared `_lastSent`** — `BusinessSync.cs:62`. A late-join re-seeds the change-detector baseline → an unswept business delta is suppressed to *other* clients. **Fix:** use a separate baseline dict for the per-join snapshot.
- **L4 · `OwnerOpenByAddress` in-memory only** — `BusinessSync.cs:37`. Not persisted / not force-re-pushed after re-host → transient recurrence of the "can't enter friend's shop" bug until the next schedule transition. **Fix:** force owners to re-push open-state on reconnect, or persist it in the manifest.
- **L5 · `ApplyBuildingVacated` clears owner/availability before the guard** — `GameStatePatcher.cs:96`. `AvailableForRent=true` + owner-clear run *before* the `!RentedByPlayer` guard → a stale/misrouted vacate flips a building the receiver actively rents. **Fix:** move both writes inside the existing guard.
- **L6 · `GetShelfFillState` reports player shops empty before stock syncs** — `MPPatches.cs:1829`. For un-entered remote shops, `itemInstances` stays empty → forced `0f` drops their real prices from the local price floor (per-machine divergence). **Fix:** gate on `IsReplicatedInterior(addr)` (the sibling shelf patch already does).
- **L7 · MPSale can charge the buyer if the host link drops mid-beat** — `MPPatches.cs:3290`. The deferred money-beat debits locally then sends `RemoteSale`; no connection check → on a link drop the send no-ops but the debit stands (no rollback). **Fix:** require the link live at send time before debiting.
- **L8 · Loan ledger keyed by mutable PlayerId** — `MPHub.cs:297`. Persisted loans keyed by character name (not StableId); a rename between sessions freezes the loan (never drafts/repays). **Fix:** key by StableId, resolve to PlayerId only for display/presence.
- **L9 · Appearances not replayed on (re)connect** — `MPServer.cs:1523`. Appearance is host-authoritative but absent from `SendJoinReplayTo`; a (re)joiner sees default-avatar capsules, possibly forever. **Fix:** peer-targeted `SendAppearanceSyncTo(peer)` from `SendJoinReplayTo`.
- **L10 · `RegisterCashier` no ownership check** — `MPServer.cs:1130`. Verifies sender identity but not shop ownership → a client forces a working cashier into an AI/other shop for the live session (no persistence). **Fix:** require sender owns/rents the address after `SenderIs`.
- **L11 · `MPAudit.Reset()` never called** — `MPAudit.cs:329`. Now that `[Audit]` is Release-enabled, stale per-session divergence streaks pollute the bug-report log signal. **Fix:** wire `Reset()` into the session-reset path.
